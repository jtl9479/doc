## 생성자란 무엇인가?

### 핵심 개념
생성자(Constructor)는 **객체가 생성될 때 자동으로 호출되는 특별한 메서드**입니다. 객체의 초기 상태를 설정하는 역할을 합니다.

### 생성자의 특징
```java
public class User {
    String name;
    int age;

    // 생성자: 클래스 이름과 동일, 반환 타입 없음
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// 객체 생성 시 생성자 호출
User user = new User("김철수", 25);
```

**생성자의 4가지 특징:**
1. **클래스 이름과 동일**: 메서드 이름이 클래스 이름과 같아야 함
2. **반환 타입 없음**: void도 쓰지 않음
3. **new 키워드와 함께 호출**: 객체 생성 시 자동 실행
4. **오버로딩 가능**: 매개변수가 다른 여러 생성자 정의 가능

### 기본 생성자 (Default Constructor)
```java
public class Product {
    String name;
    int price;

    // 기본 생성자 (매개변수 없음)
    public Product() {
        this.name = "미정";
        this.price = 0;
    }
}

// 사용
Product p = new Product();  // 기본값으로 초기화됨
```

**중요:** 생성자를 하나도 정의하지 않으면 컴파일러가 자동으로 기본 생성자를 만들어줍니다.

```java
// 생성자를 명시하지 않은 경우
public class Student {
    String name;
    int grade;
}

// 컴파일러가 자동으로 추가:
// public Student() { }

Student s = new Student();  // 가능
```

**기본 생성자 자동 생성 규칙:**
```
[Java 컴파일러의 기본 생성자 자동 생성 규칙]

1. 생성자를 하나도 정의하지 않은 경우:
   → 컴파일러가 자동으로 기본 생성자 추가

   public class User {
       String username;
   }

   // 컴파일러가 자동 추가:
   // public User() { }


2. 생성자를 하나라도 정의한 경우:
   → 컴파일러가 기본 생성자를 추가하지 않음!

   public class User {
       public User(String username) { ... }
   }

   // 기본 생성자 없음!
   // new User() 사용 불가 ❌
```

### 생성자 오버로딩
```java
public class Account {
    String accountNumber;
    String owner;
    long balance;

    // 생성자 1: 계좌번호만
    public Account(String accountNumber) {
        this.accountNumber = accountNumber;
        this.owner = "미정";
        this.balance = 0;
    }

    // 생성자 2: 계좌번호 + 소유자
    public Account(String accountNumber, String owner) {
        this.accountNumber = accountNumber;
        this.owner = owner;
        this.balance = 0;
    }

    // 생성자 3: 모든 필드
    public Account(String accountNumber, String owner, long balance) {
        this.accountNumber = accountNumber;
        this.owner = owner;
        this.balance = balance;
    }
}

// 다양한 방식으로 객체 생성 가능
Account acc1 = new Account("1001");
Account acc2 = new Account("1002", "김영희");
Account acc3 = new Account("1003", "이민수", 100000);
```

**생성자 오버로딩 규칙:**
```
✅ 구분 기준:
1. 매개변수의 개수
2. 매개변수의 타입
3. 매개변수의 순서 (타입이 다를 때)

❌ 구분 기준 아님:
1. 매개변수의 이름
2. 반환 타입 (생성자는 반환 타입 없음)
```

### this() - 생성자 체이닝
```java
public class Member {
    String memberId;
    String name;
    String email;
    int age;

    // 메인 생성자
    public Member(String memberId, String name, String email, int age) {
        this.memberId = memberId;
        this.name = name;
        this.email = email;
        this.age = age;
    }

    // 간단한 생성자들이 메인 생성자 호출
    public Member(String memberId, String name) {
        this(memberId, name, "미등록", 0);  // 다른 생성자 호출
    }

    public Member(String memberId, String name, String email) {
        this(memberId, name, email, 0);
    }
}

// 사용
Member m1 = new Member("M001", "박철수", "park@email.com", 30);
Member m2 = new Member("M002", "최영희");  // this()로 메인 생성자 호출
```

**this() 사용 규칙:**
```
1. ✅ this()는 생성자의 첫 번째 줄에 있어야 함
2. ✅ 한 생성자에서 this() 또는 super() 중 하나만 가능
3. ✅ this()로 순환 참조하면 컴파일 에러
4. ✅ this() 이후에 추가 초기화 로직 작성 가능
```

**this() 예시:**
```java
public class Book {
    String title;
    String author;
    int price;

    // 메인 생성자
    public Book(String title, String author, int price) {
        this.title = title;
        this.author = author;
        this.price = price;
        System.out.println("메인 생성자 호출");
    }

    // 간단한 생성자
    public Book(String title) {
        this(title, "미상", 0);  // 메인 생성자 호출
        System.out.println("간단한 생성자 호출 완료");
    }
}
```

## 실생활 비유로 이해하기

### 비유 1: 자동차 공장 🚗

**상황:** 자동차 공장에서 차량 생산

```
생성자 = 자동차 조립 라인
- 새 차를 만들 때마다 거치는 필수 과정
- 엔진 장착, 색상 도색, 옵션 설치 등

기본 생성자 = 기본형 차량
- 기본 옵션만 장착 (블랙, 기본 휠, 기본 시트)

오버로딩된 생성자 = 옵션별 차량
- 생성자 1: 색상만 선택
- 생성자 2: 색상 + 휠 선택
- 생성자 3: 풀옵션 (색상 + 휠 + 선루프 + 가죽시트)
```

**코드로 표현:**
```java
public class Car {
    String color;
    String wheelType;
    boolean hasSunroof;
    boolean hasLeatherSeats;

    // 기본형 (기본 옵션)
    public Car() {
        this.color = "블랙";
        this.wheelType = "기본 휠";
        this.hasSunroof = false;
        this.hasLeatherSeats = false;
        System.out.println("기본형 차량 생산 완료");
    }

    // 색상 선택형
    public Car(String color) {
        this();  // 기본 설정 먼저 적용
        this.color = color;
        System.out.println(color + " 차량 생산 완료");
    }

    // 색상 + 휠 선택형
    public Car(String color, String wheelType) {
        this(color);
        this.wheelType = wheelType;
        System.out.println(color + " + " + wheelType + " 차량 생산 완료");
    }

    // 풀옵션
    public Car(String color, String wheelType, boolean hasSunroof, boolean hasLeatherSeats) {
        this.color = color;
        this.wheelType = wheelType;
        this.hasSunroof = hasSunroof;
        this.hasLeatherSeats = hasLeatherSeats;
        System.out.println("풀옵션 " + color + " 차량 생산 완료");
    }

    void displayCarInfo() {
        System.out.println("\n=== 차량 정보 ===");
        System.out.println("색상: " + color);
        System.out.println("휠: " + wheelType);
        System.out.println("선루프: " + (hasSunroof ? "있음" : "없음"));
        System.out.println("가죽시트: " + (hasLeatherSeats ? "있음" : "없음"));
        System.out.println("================\n");
    }
}

// 공장 가동
public class CarFactory {
    public static void main(String[] args) {
        System.out.println("=== 자동차 공장 가동 ===\n");

        Car car1 = new Car();  // 기본형
        car1.displayCarInfo();

        Car car2 = new Car("화이트");  // 색상 선택
        car2.displayCarInfo();

        Car car3 = new Car("레드", "스포츠 휠");  // 색상+휠
        car3.displayCarInfo();

        Car car4 = new Car("블루", "크롬 휠", true, true);  // 풀옵션
        car4.displayCarInfo();
    }
}
```

**핵심 교훈:**
- 생성자 = 제품 조립 라인
- 기본 생성자 = 기본 사양
- 오버로딩 = 다양한 옵션 패키지
- this() 체이닝 = 기본 조립 후 추가 옵션 장착

### 비유 2: 스마트폰 초기 설정 📱

**상황:** 새 스마트폰 개봉 후 초기 설정

```
생성자 = 스마트폰 첫 실행 시 초기 설정 마법사
- 언어 선택
- Wi-Fi 연결
- Google/Apple 계정 로그인
- 기본 앱 설치

생성자 없이 객체 생성 = 설정 없이 폰 사용
→ 불가능! 반드시 초기 설정 필요

생성자 오버로딩 = 설정 옵션
- 빠른 설정 (기본값으로 자동 설정)
- 상세 설정 (모든 항목 직접 선택)
```

**코드로 표현:**
```java
public class Smartphone {
    String brand;
    String model;
    String language;
    String accountEmail;
    boolean isWifiConnected;
    int batteryLevel;

    // 빠른 설정 생성자 (기본값 사용)
    public Smartphone(String brand, String model) {
        this.brand = brand;
        this.model = model;
        this.language = "한국어";  // 기본값
        this.accountEmail = null;  // 나중에 설정
        this.isWifiConnected = false;
        this.batteryLevel = 50;

        performInitialSetup();
    }

    // 상세 설정 생성자 (모든 옵션 지정)
    public Smartphone(String brand, String model, String language,
                      String accountEmail, boolean isWifiConnected) {
        this.brand = brand;
        this.model = model;
        this.language = language;
        this.accountEmail = accountEmail;
        this.isWifiConnected = isWifiConnected;
        this.batteryLevel = 50;

        performInitialSetup();
    }

    // 중고폰 복원 생성자 (배터리 상태 포함)
    public Smartphone(String brand, String model, String language,
                      String accountEmail, int batteryLevel) {
        this.brand = brand;
        this.model = model;
        this.language = language;
        this.accountEmail = accountEmail;
        this.isWifiConnected = false;
        this.batteryLevel = batteryLevel;

        System.out.println("중고폰 복원 중...");
        performInitialSetup();
    }

    private void performInitialSetup() {
        System.out.println("\n=== " + brand + " " + model + " 초기 설정 시작 ===");
        System.out.println("언어: " + language);
        System.out.println("계정: " + (accountEmail != null ? accountEmail : "미설정"));
        System.out.println("Wi-Fi: " + (isWifiConnected ? "연결됨" : "연결 안 됨"));
        System.out.println("배터리: " + batteryLevel + "%");
        System.out.println("기본 앱 설치 완료");
        System.out.println("=== 설정 완료! 사용 가능 ===\n");
    }
}
```

### 비유 3: 은행 계좌 개설 🏦

**상황:** 은행에서 계좌 개설할 때

```
생성자 = 계좌 개설 절차
- 필수 정보: 이름, 주민등록번호, 계좌 종류
- 선택 정보: 초기 입금액, 비밀번호

기본 생성자 = 기본 계좌 개설
- 보통예금, 초기 잔액 0원, 임시 비밀번호

맞춤형 생성자 = 다양한 계좌 개설 옵션
- 적금 계좌 (월 납입액 지정)
- 대출 계좌 (대출 한도 지정)
```

**코드로 표현:**
```java
public class BankAccount {
    String accountNumber;
    String ownerName;
    String accountType;
    long balance;
    String pin;

    // 기본 보통예금 개설
    public BankAccount(String ownerName) {
        this.accountNumber = generateAccountNumber();
        this.ownerName = ownerName;
        this.accountType = "보통예금";
        this.balance = 0;
        this.pin = "0000";  // 임시 비밀번호

        printAccountInfo("기본 계좌 개설 완료");
    }

    // 초기 입금액 포함 계좌 개설
    public BankAccount(String ownerName, long initialDeposit) {
        this(ownerName);  // 기본 생성자 호출
        this.balance = initialDeposit;

        System.out.println("초기 입금액: " + initialDeposit + "원");
    }

    // 특정 계좌 타입 개설 (적금, 정기예금 등)
    public BankAccount(String ownerName, String accountType, long initialDeposit, String pin) {
        this.accountNumber = generateAccountNumber();
        this.ownerName = ownerName;
        this.accountType = accountType;
        this.balance = initialDeposit;
        this.pin = pin;

        printAccountInfo(accountType + " 계좌 개설 완료");
    }

    private String generateAccountNumber() {
        return "110-" + (System.currentTimeMillis() % 1000000);
    }

    private void printAccountInfo(String message) {
        System.out.println("\n━━━━━━━━━━━━━━━━━━━━━━━━━━");
        System.out.println(message);
        System.out.println("━━━━━━━━━━━━━━━━━━━━━━━━━━");
        System.out.println("계좌번호: " + accountNumber);
        System.out.println("예금주: " + ownerName);
        System.out.println("계좌종류: " + accountType);
        System.out.println("잔액: " + balance + "원");
        System.out.println("━━━━━━━━━━━━━━━━━━━━━━━━━━\n");
    }
}
```

### 비유 4: 신분증 발급 💳

**상황:** 주민등록증이나 운전면허증 발급

```
생성자 = 신분증 발급 프로세스
- 필수: 이름, 생년월일, 주소
- 선택: 사진, 서명

생성자 호출 시점 = 신분증 발급 신청 순간
- 한 번 발급되면 정보 변경 어려움 (재발급 필요)
- 발급 시 모든 정보를 정확히 입력해야 함

this() 체이닝 = 신분증 종류별 공통 절차
- 모든 신분증은 기본 정보 입력 → 추가 정보 입력
```

**코드로 표현:**
```java

        this.name = name;
        this.birthDate = birthDate;
        this.address = address;
        this.photoPath = photoPath;

        System.out.println("║  " + cardType + "                 ║");
```

### 비유 5: 아파트 입주 🏢

**상황:** 새 아파트 입주 시 초기 설정

```
생성자 = 입주 절차
- 필수: 동/호수, 입주자 이름, 계약 유형
- 선택: 가구 옵션, 주차 여부, 반려동물

기본 생성자 = 기본 입주 (빈집 상태)
생성자 오버로딩 = 입주 옵션 패키지
- 패키지 A: 기본 가구 포함
- 패키지 B: 풀옵션 가구 + 주차
- 패키지 C: 맞춤 설정
```

**코드로 표현:**
```java
public class Apartment {
    String dong;
    String ho;
    String ownerName;
    String contractType;
    boolean hasFurniture;
    boolean hasParking;
    boolean hasPet;
    int monthlyFee;

    // 기본 입주 (빈집)
    public Apartment(String dong, String ho, String ownerName) {
        this.dong = dong;
        this.ho = ho;
        this.ownerName = ownerName;
        this.contractType = "월세";
        this.hasFurniture = false;
        this.hasParking = false;
        this.hasPet = false;
        this.monthlyFee = 500000;

        performMoveIn();
    }

    // 패키지 A: 기본 가구 포함
    public Apartment(String dong, String ho, String ownerName, boolean hasFurniture) {
        this(dong, ho, ownerName);  // 기본 입주 먼저
        this.hasFurniture = hasFurniture;
        if (hasFurniture) {
            this.monthlyFee += 100000;
            System.out.println("가구 옵션 추가: +100,000원");
        }
    }

    // 패키지 B: 풀옵션
    public Apartment(String dong, String ho, String ownerName,
                     boolean hasFurniture, boolean hasParking) {
        this(dong, ho, ownerName, hasFurniture);
        this.hasParking = hasParking;
        if (hasParking) {
            this.monthlyFee += 50000;
            System.out.println("주차 옵션 추가: +50,000원");
        }
    }

    // 패키지 C: 완전 맞춤 설정
    public Apartment(String dong, String ho, String ownerName, String contractType,
                     boolean hasFurniture, boolean hasParking, boolean hasPet) {
        this.dong = dong;
        this.ho = ho;
        this.ownerName = ownerName;
        this.contractType = contractType;
        this.hasFurniture = hasFurniture;
        this.hasParking = hasParking;
        this.hasPet = hasPet;

        // 계약 타입별 기본 월세
        if (contractType.equals("전세")) {
            this.monthlyFee = 0;
        } else if (contractType.equals("월세")) {
            this.monthlyFee = 500000;
        }

        // 옵션별 추가 비용
        if (hasFurniture) this.monthlyFee += 100000;
        if (hasParking) this.monthlyFee += 50000;
        if (hasPet) this.monthlyFee += 30000;

        performMoveIn();
    }

    private void performMoveIn() {
        System.out.println("\n╔═══════════════════════════╗");
        System.out.println("║     아파트 입주 완료      ║");
        System.out.println("╠═══════════════════════════╣");
        System.out.println("║ 위치: " + dong + "동 " + ho + "호");
        System.out.println("║ 입주자: " + ownerName);
        System.out.println("║ 계약: " + contractType);
        System.out.println("║ 가구: " + (hasFurniture ? "있음" : "없음"));
        System.out.println("║ 주차: " + (hasParking ? "가능" : "불가"));
        System.out.println("║ 반려동물: " + (hasPet ? "있음" : "없음"));
        System.out.println("║ 월세: " + monthlyFee + "원");
        System.out.println("╚═══════════════════════════╝\n");
    }
}
```

## 정리

### 생성자의 핵심 포인트

**1. 생성자의 역할**
- 객체 생성 시 자동 호출
- 필드 초기화
- 객체의 올바른 초기 상태 보장

**2. 생성자 vs 일반 메서드**
```java
// 생성자
public User(String name) {  // 클래스 이름과 동일
    this.name = name;
}

// 일반 메서드
public void setName(String name) {  // 반환 타입 있음
    this.name = name;
}
```

**3. 언제 생성자를 사용하는가?**
- ✅ 객체 생성 시 필수 값 설정
- ✅ 유효성 검사가 필요한 초기화
- ✅ 복잡한 초기화 로직
- ✅ 불변 객체 생성 (final 필드 초기화)

**4. 실무 팁**
```java
// 권장: 메인 생성자 하나 + this() 체이닝
public class Product {
    // 메인 생성자 (모든 초기화 로직)
    public Product(String name, int price, int stock) {
        this.name = name;
        this.price = price;
        this.stock = stock;
    }

    // 간단한 생성자들
    public Product(String name, int price) {
        this(name, price, 0);  // 메인 생성자 재사용
    }

    public Product(String name) {
        this(name, 0, 0);
    }
}
```
